CONCURRENCIA

Definicion

Acción de coincidir distintas personas, sucesos, privilegios o cosas en un mismo lugar o tiempo.

Definicion de Concurrencia en Programacion

La concurrencia en JavaScript se refiere a la capacidad de ejecutar múltiples tareas de manera simultánea o en paralelo para mejorar la eficiencia y el rendimiento. JavaScript utiliza un modelo de concurrencia basado en eventos, principalmente a través de su event loop (bucle de eventos)

Event Loop

El event loop es un mecanismo que permite a JavaScript realizar operaciones no bloqueantes, a pesar de ser un lenguaje de un solo hilo. Funciona de la siguiente manera:

  • Call Stack (Pila de Llamadas): Donde se ejecuta el código. Es una estructura de datos LIFO (Last In, First Out).
  • Task Queue (Cola de Tareas): Donde se colocan las tareas que deben ejecutarse después de que el call stack esté vacío.
  • Microtask Queue (Cola de Microtareas): Similar a la task queue, pero con una prioridad más alta. Aquí se colocan las microtareas, como las resoluciones de promises.

El event loop continuamente verifica si el call stack está vacío. Si está vacío, toma la primera tarea de la task queue o microtask queue y la coloca en el call stack para su ejecución.
